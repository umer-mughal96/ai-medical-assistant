# AI-Based Medical Assistant

A full-stack medical assistant application that enables doctors to manage patients, create medical cases, and interact with an AI assistant (ChatGPT) for diagnosis and treatment recommendations. The system automatically saves treatment plans generated by the AI.

## 🎯 Project Overview

This application provides a comprehensive medical case management system with the following features:

- **Doctor Authentication**: Secure registration and login with JWT tokens
- **Patient Management**: Create, view, and manage patient records
- **Case Management**: Create and manage medical cases for each patient
- **AI-Powered Diagnosis**: Interactive chat interface with ChatGPT for medical consultations
- **Auto-Saved Treatment Plans**: AI-generated treatment recommendations are automatically saved to the database
- **Persistent Chat History**: All conversations between doctors and AI are saved and can be retrieved

## 🏗️ Architecture & Approach

### Tech Stack

**Backend:**
- Node.js with Express.js
- PostgreSQL database with Prisma ORM
- JWT for authentication
- OpenAI API integration for AI chat
- bcryptjs for password hashing

**Frontend:**
- React 19 with Vite
- React Router for navigation
- Axios for API calls
- Modern CSS with responsive design

### System Design

1. **Authentication Flow**: Doctors register/login → receive JWT token → token stored in localStorage → used for all authenticated requests
2. **Data Model**: Doctor → Patients → Cases → Messages (chat history)
3. **AI Integration**: Each message sent to AI includes full chat history for context-aware responses
4. **Cascade Deletion**: Proper handling of foreign key constraints when deleting patients/cases

## 📋 Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** (v18 or higher)
- **npm** or **yarn**
- **PostgreSQL** (v12 or higher)
- **OpenAI API Key** (for AI chat functionality)

## 🚀 Setup Instructions

### 1. Clone the Repository

```bash
git clone <repository-url>
cd task
```

### 2. Backend Setup

```bash
cd backend
npm install
```

### 3. Database Configuration

1. Create a PostgreSQL database:
```bash
createdb medical_assistant
# Or using psql:
# psql -U postgres
# CREATE DATABASE medical_assistant;
```

2. Create a `.env` file in the `backend` directory:
```bash
cd backend
touch .env
```

3. Add the following environment variables to `backend/.env`:
```env
DATABASE_URL="postgresql://username:password@localhost:5432/medical_assistant"
JWT_SECRET="your-super-secret-jwt-key-change-this-in-production"
OPENAI_API_KEY="your-openai-api-key-here"
PORT=4000
```

**Important:** Replace:
- `username` and `password` with your PostgreSQL credentials
- `your-super-secret-jwt-key-change-this-in-production` with a strong random string
- `your-openai-api-key-here` with your actual OpenAI API key

### 4. Run Database Migrations

```bash
cd backend
npm run prisma:migrate
```

This will create all necessary tables in your database.

### 5. Start the Backend Server

```bash
cd backend
npm run dev
# Or for production:
# npm start
```

The backend will run on `http://localhost:4000`

### 6. Frontend Setup

Open a new terminal window:

```bash
cd frontend
npm install
```

### 7. Start the Frontend Development Server

```bash
cd frontend
npm run dev
```

The frontend will run on `http://localhost:5173` (or another port if 5173 is busy)

## 🎮 Using the Application

### 1. Register/Login

1. Navigate to `http://localhost:5173`
2. Click "No account? Register" to create a new doctor account
3. Fill in your name, email, and password
4. After registration, you'll be automatically logged in

### 2. Manage Patients

- Click "Patients" in the navigation
- Use the form at the top to create new patients
- View all patients in the table below
- Click "View Cases" to manage cases for a specific patient
- Click "Delete" to remove a patient (this will also delete all associated cases)

### 3. Manage Cases

- From the Patients page, click "View Cases" for any patient
- Create a new case by entering a title
- View all cases in the table
- Click "Edit" to modify case title or status (open/closed)
- Click "Open Chat" to start a diagnosis session

### 4. AI Diagnosis Chat

- In the chat interface, describe patient symptoms or ask questions
- The AI will respond with diagnosis and treatment recommendations
- All messages (doctor and AI) are saved to the database
- The latest AI response is automatically saved as the treatment plan
- Chat history persists across page reloads

## 📡 API Endpoints

### Authentication
- `POST /api/auth/register` - Register a new doctor
- `POST /api/auth/login` - Login and receive JWT token

### Patients
- `GET /api/patients` - Get all patients for logged-in doctor
- `POST /api/patients` - Create a new patient
- `DELETE /api/patients/:id` - Delete a patient (cascades to cases and messages)

### Cases
- `GET /api/cases/:patientId` - Get all cases for a patient
- `POST /api/cases/:patientId` - Create a new case
- `PATCH /api/cases/:caseId` - Update case (title, status)
- `DELETE /api/cases/:caseId` - Delete a case (cascades to messages)
- `GET /api/cases/messages/:caseId` - Get all chat messages for a case
- `POST /api/cases/ai/:caseId` - Send message to AI and get response

## 🔒 Security Features

- Password hashing with bcryptjs
- JWT token-based authentication
- Protected routes (frontend and backend)
- Authorization checks ensure doctors can only access their own patients/cases
- CORS enabled for frontend-backend communication

## 🗄️ Database Schema

- **Doctor**: Stores doctor credentials and profile
- **Patient**: Linked to doctor, contains patient information
- **Case**: Medical cases linked to patients
- **Message**: Chat messages between doctor and AI, linked to cases

## 📝 Notes

- The application uses PostgreSQL as the database. Make sure PostgreSQL is running before starting the backend.
- OpenAI API key is required for the AI chat functionality. Without it, the chat feature will not work.
- JWT tokens expire after 1 day. Users will need to log in again after expiration.
- All delete operations properly handle foreign key constraints with cascade deletion.
